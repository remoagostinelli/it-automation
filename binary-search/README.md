# Binary Searching a Problem
Binary searching is really efficient when trying to find an element in a sorted list.

In troubleshooting, we can apply this idea when we
need to go through and test a long list of hypotheses.
When doing this, the list of elements contains all the possible causes of the problem and we keep reducing the problem by
half until only one option is left.

The list of elements could be entries in a file, extensions enabled, boards connected to a server, or even lines of code added to a faulty release. With each iteration, the problem is cut in half.
